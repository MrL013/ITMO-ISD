# Этап 1. Сценарии и первичная архитектура AgroVision

## 1) Цель сервиса
`AgroVision` помогает агроному и оператору хозяйства принимать решения по поливу, обследованию дронами и планированию работ на основании:
- телеметрии IoT-датчиков;
- наблюдений/облетов дронов;
- аналитики и рекомендаций от LLM.

## 2) Ключевые роли
- Агронóm: анализирует состояние поля, утверждает действия.
- Оператор: контролирует телеметрию и запускает рекомендации.
- Руководитель хозяйства: смотрит KPI ресурсов и урожайности.
- Инженер IoT/дронов: обслуживает сенсоры, каналы связи и миссии.

## 3) Основные сценарии
1. Мониторинг поля в реальном времени:
- датчики передают влажность, температуру, индексы растений;
- система формирует сводку по полю.

2. Формирование рекомендаций LLM:
- оператор запрашивает совет на 24 часа;
- LLM получает контекст (культура, площадь, метрики), возвращает текст и действия.

3. Снижение расхода ресурсов:
- система рекомендует точечный полив и перенос работ по погодному окну;
- ожидаемый результат: меньше воды/топлива и меньше внеплановых выездов.

4. Инцидентный сценарий:
- при аномалиях (низкая влажность, стресс растений) система создает приоритетное действие для оперативной реакции.

## 4) Первичная архитектура
1. Edge-слой:
- сбор показаний с датчиков через шлюзы/контроллеры;
- фильтрация/агрегация перед отправкой в облако.

2. Cloud backend:
- API-шлюз (`REST`);
- сервис телеметрии;
- сервис аналитики;
- сервис LLM-рекомендаций;
- сервис задач (полив, миссии дронов).

3. Интеграционный слой:
- брокер сообщений для событийной передачи (`telemetry.received`, `anomaly.detected`, `recommendation.generated`).

4. Клиенты:
- web/mobile панель для агронома и оператора.

## 5) Почему LLM в контуре
LLM используется как слой интерпретации и объяснимых рекомендаций:
- превращает сырые метрики в понятный план действий;
- учитывает агрономический контекст (тип культуры, текущее состояние);
- дает компактный список операций для персонала.

## 6) Ограничения прототипа
- хранилище in-memory (без БД);
- визуализация без картографического движка;
- интеграция с OpenAI включается только при наличии `OPENAI_API_KEY`, иначе fallback-логика.
